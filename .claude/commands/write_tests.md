Napisz kompleksowe testy dla: $ARGUMENTS
Konwencje testowania
- Używaj Vitest wraz z React Testing Library
- Umieszczaj pliki testów w katalogu tests w tym samym folderze co plik źródłowy
- Nazywaj pliki testów jako [filename].test.ts(x)
- Używaj prefiksu @/ do importów
Zakres testów
- Testuj scenariusze pozytywne (happy paths)
- Testuj przypadki brzegowe
- Testuj stany błędów
- Skupiaj się na testowaniu zachowania i publicznego API, a nie szczegółów implementacyjnych
